FHIR 概述（面向患者）

[%wgt pe%] 工作组成熟度: N/A标准状态:Informative

欢迎！您已导航到 FHIR （快速卫生信息交互资源）标准的患者介绍。如果您最近听说了与个人健康记录相关的一种叫做 "fire" 的东西，那么您来对地方了。以下文档将简要概述作为患者，您如何根据各种政府法律要求访问自己的个人健康记录。

快速卫生信息交互资源（FHIR）是一种用于共享医疗数据的数据规范。与以前基于传真机或 PDF 文档的标准不同，FHIR 格式基于现代 Web 标准。从根本上说，它是“医疗”和“互联网”的结合。为了使这成为可能，医疗系统和技术供应商必须聚在一起，就如何相互发送和共享数据达成一致。我们称此为应用程序编程接口（API）；我们使用的协议是超文本传输协议（HTTP）。因此，FHIR 规范是一个关于如何通过 Web 上的超文本（HTTP）发送医疗数据的 API 规范。

许多医疗系统和技术供应商已经在支持 FHIR 数据标准，包括美国最大医院的电子病历系统，以及苹果和微软等主要技术供应商。

FHIR 如何帮助作为患者的我？

掌握自己的健康记录可以是一个可以改变自身生活并赋予自己力量的决定。与其猜测您自己的健康状况，您可以参与测量和了解您健康状况的过程 。在以下情况时，您会发现 FHIR 很有用：

  • 从不同的诊所汇总您的健康记录。
  • 使用应用程序、传感器、可穿戴设备和其他设备收集和跟踪健康数据。
  • 研究和了解您自己的健康状况。
  • 研究和了解您的医疗保健提供者做出的诊断。
  • 确保您的医生或护士关于您的记录是正确的。
  • 与医疗团队共同参与制定和执行您的护理计划或治疗计划。
  • 与您的护理团队分享您收集的数据。
  • 与朋友、家人和看护人分享您的记录。
  • 在专科会诊时分享您的记录。
  • 计划外科手术。
  • 从您的付款人处下载账单记录。
  • 从一个城市搬到另一个城市。

如何使用 FHIR？

事实上，您可能已经在不知不觉中使用了 FHIR 。当医院系统在不同环境之间传输记录时，经常使用FHIR。这可能包括您用于访问记录的患者门户网站；您已注册的健康信息交换系统；或您智能手机上的个人健康记录应用程序，例如 Apple Health 或 Android Common Health……

很多时候，数据的互操作正在发生，但它在幕后工作。然而，懂行（savvy）的患者可以直接访问 FHIR 端点。为此，患者需要一个 FHIR 应用程序（手机上的应用程序或在线应用程序）；由您的医疗保健机构提供的用户凭据；以及可能的注册过程，将应用程序注册到医疗保健系统中。

为什么我需要我的记录副本？

患者调查显示，患者倾向于希望拥有自己健康记录副本的一个主要原因是检查其准确性和错误。当病历中存在错误时，可能会导致严重的安全问题。最常见的错误包括左右颠倒和病历中包含其他患者的内容。获取您的病历副本可以帮助确保您的病历准确无误。

患者经常在希望更积极地管理自己的健康时要求获取自己的病历副本。这通常发生在健康事件之后，个人会重新评估自己的习惯并寻求实施新的护理计划和新的日常例行（routine）。这可以简单到追踪个人的健身目标，也可以复杂到进行全面的量化自我检查，并每月或每季度测量代谢生物标志物。

我如何获取我的记录副本？

健康记录可从许多组织获得，例如医院、诊所、保险公司、实验室、医疗设备供应商等。在大多数情况下，您的记录将通过密码保护在您的账户中，并受各种隐私法律的约束。这种隐私保护正是它们通常具有受限访问权限的原因，也是为什么患者访问这些记录成为一个关注的问题。

通常，首先需要与当地的医院、诊所或实验室联系，看看他们是否有患者门户。如果有，注册并查找可下载的数据。这些文件一般以 PDF 格式提供，但您还应查找“导出”或“下载”按钮。

其次，像这样的消费者健康信息交换平台会为用户提供可浏览的参与提供者目录，用户可以从这些提供者处下载他们的数据。

患者有哪些法律权利？

许多法律涵盖了患者隐私、患者数据所有权以及患者访问自身数据的权利。相关法律和法律先例包括：

  • 健康保险流通和责任法案 (美国)
  • 医疗保险访问和 CHIP 重新授权法案 (美国)
  • 21世纪治愈法案 (美国)
  • 21世纪治愈法案 (信息封锁规则)
  • 冠状病毒援助、救济和经济安全法案] (美国)
  • 通用数据保护条例 (欧洲)
  • 个人信息保护法 (中国)
  • 隐私法 (澳大利亚)

请注意，在美国，《21世纪治愈法案》对患者尤其重要；该法规包括条款，要求患者可以免费通过电子方式获取其所有的电子健康信息（EHI），无论是结构化还是非结构化信息，并且可以使用他们选择的应用程序。患者有权向报告信息阻塞由医疗系统和其他供应商造成的情况。

包含什么数据？

FHIR 为医疗保健中常见的许多数据类型提供了现成的数据模式和实现指导……过敏、免疫、药物、患者人口统计信息、临床观察、生命体征等，示例包括如下：

  • 美国互操作性通用数据 (USCDI)
    USCDI 是美国政府期望应用程序和技术供应商支持的一组基本数据，以获得作为互操作性合规产品或服务的认证。

  • 阿戈诺特项目 阿戈诺特项目（这个名称是对JSON格式的一种双关语，指的是荷马史诗《奥德赛》中的伊阿宋和阿戈诺特人）是最早的FHIR项目之一，包含了一组核心数据资源，EHR供应商已同意使用这些资源来传达临床发现。

  • 国际患者摘要
    国际患者摘要是由欧盟主导的项目，专注于在欧盟不同国家之间交换的患者摘要。特别关注不同的语言和术语，以及跨境所需的必要文件（如疫苗接种）。

  • 疫苗卡
    公共卫生数据，例如疫苗护照和疫苗接种状态。

  • 连续护理文件 (CCD on FHIR) 连续护理文件是您在医院过夜留院后，医院和诊所提供给您的临床文件。它们包括病程记录、治疗总结、诊断等文件。还可能包括救护车和运输指示。

  • 支付方数据，例如财务索赔和利益解释文件，以及他们可能拥有的临床数据。
